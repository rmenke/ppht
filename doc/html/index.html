Introduction

PPHT is a header-only library that implements the Probabilistic Progressive Hough Transform (PPHT) Line Segment Detection algorithm. This library uses the dynamic threshold approach proposed by the authors of the original paper.

About the Algorithm

The formula for a line \(L\) passing through the point \((x_0, y_0)\) is \(\Delta y(x-x_0) = \Delta x(y-y_0).\) Unlike the traditional slope-intercept form, this equation handles vertical lines. Assume \((x_0, y_0)\) is the point where the perpendicular \(P\) to the line that passes through the origin intersects \(L.\) \(P\) makes an angle \(\theta\) with the x-axis; therefore, the angle \(L\) makes with the axis is \(\theta+\frac{\pi}{2}.\) From this, we get \(\Delta y=\sin(\theta+\frac{\pi}{2})=\cos\theta\) and \(\Delta x=\cos(\theta+\frac{\pi}{2})=-\sin\theta.\) We can also let \(x_0=\rho\cos\theta\) and \(y_0=\rho\sin\theta\) for some unknown \(\rho.\) Plugging these values into the original equation and solving for \(\rho\) yields:

\begin{align} 0 &= \Delta y(x-x_0) - \Delta x(y-y_0)\\ &= (x-x_0)\cos\theta + (y-y_0)\sin\theta\\ &= (x-\rho\cos\theta)\cos\theta + (y-\rho\sin\theta)\sin\theta\\ &= x\cos\theta - \rho\cos^2\theta + y\sin\theta - \rho\sin^2\theta\\ &= x\cos\theta + y\sin\theta - \rho\\ \rho &= x\cos\theta + y\sin\theta \end{align}

This implies that every line can be defined by the pair \((\theta,\rho)\) and for a given \(x\), \(y\), and \(\theta\), \(\rho\) can be calculated. If we restrict \(0\le\theta<\pi\) then every line has a unique representation as a point in a different space. Equivalently, every point in \(xy\)-space is a sinusoid in \(\theta\rho\)-space. This is known as the Hough transform. Given a set of points in \(xy\)-space, if their sinusoids intersect at a point in \(\theta\rho\)-space then that point represents the line on which all of the points are located. Locating those intersections is the key to the Hough Transform Line Detection algorithm.

The algorithm works by quantizing the \(\theta\) and \(\rho\) values, iterating over the possible values of \(\theta\) for every set pixel, finding the quantized value for \(\rho\), and incrementing a counter in an accumulator array. The peaks in this array are the lines on which a large number of pixels reside. Locating these peaks is where the algorithm has the most trouble.

Progressive Hough Transform algorithms work by randomly sampling the set pixels in a bitmap image and incrementing the counters. When a counter reaches a given threshold (either fixed or computed) the algorithm pauses, scans the line corresponding to the counter, and removes the pixels from the bitmap for any line segments found.

The Probabilistic Progressive Hough Transform (PPHT) uses combinatorics to determine the threshold for a counter to trigger a scan. It starts by assuming the null hypothesis: the image is a collection of random pixels. Since each vote increments a counter in each column \((\theta)\) of the accumulator, the expected value for any counter after \(v\) votes would be \(E[n] = \frac{v}{\rho_{\max}}\), where \(\rho_{\max}\) is the height of the counter matrix.

The simplifying assumption that we make is that the counters are selected by a uniform process. For this, a binomial distribution would be used to calculate the probability of a bin containing \(n\) after \(v\) voting rounds. However, the probability mass function for a binomial distribution is expensive to compute, so instead we approximate the probability by using the Poisson distribution instead. Setting \(\lambda = E[n]\), we get

\begin{align} P(n)&=\frac{\lambda^n}{n!}e^{-\lambda}\\ &=\frac{\lambda^n}{\Gamma(n+1)}e^{-\lambda}\\ \ln P(n)&=\ln(\lambda^n)-\ln\Gamma(n+1)-\lambda\\ &=n\cdot\ln(\lambda)-\ln\Gamma(n+1)-\lambda \end{align}

If the value \(P(n)\) is less than a predetermined threshold \(t\) – or equivalently \(\ln P(N)<\ln(t)\) – then we reject the null hypothesis and search along the line described by \((\theta,\rho)\) for a line segment.
